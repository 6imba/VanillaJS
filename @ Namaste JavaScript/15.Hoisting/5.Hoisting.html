Finnally;

  • var variable are hoisted on Global-Scope/MemorySpace/Stack but let&const variable are hoisted on Script-Scope/MemorySpace/Stack
  • variable are hoisted with value as undefined where as function&class are hoisted with value as their defination.
  • arrow function and funtion_expression hold by variable is hoisted with value as undefined as they act as normal variable
  • even though all variable(var,let,const) are hoisted, we can only invoke/access Global_Scope variable before declaration/assignment; i.e: variable declared using var keyword. Where as invoking/accessing Script_Scope variable before declaration/assignment; i.e: variable declared using let&const keyword gives ReferenceError even though they are hoisted. (Means we cannot access/invoke let&const variable before they are initialized.)
  • classes are hoisted in Script-Scope. So give ReferenceError when you try to access/invoke it before declaration.

# Variables declared with let and const are also hoisted but, unlike var , they are not hoisted in Global_Scope so we cannot access them which lead to ReferenceError.

Note: So if let & const are not hoisted in GlobalScope, where are they hoisted? ==> They are hoisted in TemporalDeadZone in the time being.(Lets Dive into TDZ)
